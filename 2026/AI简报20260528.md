## 🔥 今日精选

🔴 **[Anthropic]** [How we contain Claude across products](https://www.anthropic.com/engineering/how-we-contain-claude) — Anthropic 系统梳理了 claude.ai、Claude Code、Claude Cowork 的隔离设计：服务端 gVisor、OS sandbox、本地 VM、网络出口控制和文件挂载权限，核心判断是 Agent 的风险边界应先由环境层兜住，再由模型行为层补强。

🔴 **[Datacurve / DeepSWE]** [DeepSWE](https://deepswe.datacurve.ai/) — DeepSWE 发布面向真实软件工程任务的长程 Coding Agent 评测，包含 113 个原创任务、91 个活跃仓库和 5 种语言，强调“无污染任务 + 手写行为验证”。VentureBeat 报道称该榜单把前沿模型差距拉大，GPT-5.5 以 70% 排名第一。

🔴 **[OpenAI]** [Building self-improving tax agents with Codex](https://openai.com/index/building-self-improving-tax-agents-with-codex/) — OpenAI 与 Thrive Holdings 披露 Tax AI 的自改进闭环：会计师修正、生产 traces、定向 evals 被整理成 Codex 可执行的工程任务。这个案例的意义不在“税务自动化”本身，而在于展示了垂直行业 Agent 如何把专家反馈转成持续改进信号。

## 💡 今日洞察

今天几条核心新闻都指向同一件事：Agent 的上限不只由模型决定，而由评测、隔离、权限、traces、evals 和反馈闭环共同决定。OpenAI 展示了“生产数据如何喂回 Agent 改进”，Anthropic 展示了“Agent 能做事之前必须先被关在正确的盒子里”，DeepSWE 则提醒大家：旧的 Coding Agent 榜单可能已经不够分辨真实工程能力。

中文圈的焦点也类似，智谱的 Scaling Pain 说明长上下文、高并发 Agent 服务会把底层推理系统的状态一致性问题放大。接下来比拼的不只是“模型会不会”，而是“系统能不能稳定地让它一直会”。
