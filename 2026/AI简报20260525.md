## 🔥 今日精选

🔴 **[Google Developers]** [Supercharging LLM inference on Google TPUs: Achieving 3X speedups with diffusion-style speculative decoding](https://developers.googleblog.com/supercharging-llm-inference-on-google-tpus-achieving-3x-speedups-with-diffusion-style-speculative-decoding/)：UCSD 在 Google TPU 上实现 DFlash，用块扩散式 speculative decoding 绕开逐 token 草稿瓶颈，平均提速 3.13 倍，并已集成到 vLLM 生态。背景：推理延迟和吞吐已经变成模型商业化的硬成本。

🔴 **[The Decoder]** [Researchers let Claude Code discover AI scaling algorithms that humans probably wouldn't have designed](https://the-decoder.com/researchers-let-claude-code-discover-ai-scaling-algorithms-that-humans-probably-wouldnt-have-designed/)：UMD、Google、Meta 等研究者用 AutoTTS 让 coding agent 搜索推理控制算法，找到的方案在保持准确率的同时，比 standard self-consistency 少用约 70% 计算，搜索成本约 40 美元、耗时 160 分钟。背景：Agent 不只写业务代码，也开始参与算法发现。

🔴 **[Google Developers]** [Building with Gemini Embedding 2: Agentic multimodal RAG and beyond](https://developers.googleblog.com/building-with-gemini-embedding-2/)：Gemini Embedding 2 GA，将文本、图像、视频、音频和文档映射到统一语义空间，支持 100+ 语言和 Matryoshka 降维，面向 agentic RAG、视觉搜索和内容审核。背景：多模态检索是 Agent 理解真实业务资料的底座。

🟡 **[The Decoder]** [ByteDance study finds that asking LMMs questions beats making it transcribe text for long document training](https://the-decoder.com/bytedance-study-finds-that-asking-lmms-questions-beats-making-it-transcribe-text-for-long-document-training/)：ByteDance Seed 研究显示，对长图文文档训练 LMM 时，让模型回答问题并定位证据，比先转写全文更有效，7B 模型也能在长文档上超过更大模型。

🟡 **[Hacker News / Epoch AI]** [Memory has grown to nearly two-thirds of AI chip component costs](https://epoch.ai/data-insights/ai-chip-component-cost-shares)：HN 热帖讨论 AI 芯片成本结构，内存已接近组件成本的三分之二。推理扩张后，瓶颈不只在算力，也在显存、带宽和封装。

🟡 **[Hacker News / arXiv]** [Constraint Decay: The Fragility of LLM Agents in Back End Code Generation](https://arxiv.org/abs/2605.06445)：论文指出 LLM Agent 在后端代码生成中会逐步丢失约束，越长链路越容易偏离原始要求。对 coding agent 来说，测试、约束追踪和中途校验比单次生成更关键。

## 💡 今日洞察

今天的主线不是“又一个更强模型”，而是 AI 系统的工程化细节：推理怎么更快，Agent 怎么少浪费 token，端侧怎么接工具，检索怎么支持多模态资料。Coding Agent 的热度仍高，但 HN 上关于 constraint decay 的讨论提醒了一件事：Agent 越像员工，越需要过程约束和验收机制。中文圈的 Vibe Coding 破圈很有意思，它说明软件创造门槛在下降，但“能跑”和“能托付”仍是两件事。
