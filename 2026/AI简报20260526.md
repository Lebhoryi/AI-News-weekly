## 🔥 今日精选

🔴 **[The Decoder]** [Google DeepMind's AlphaProof Nexus solves decades-old math problems for a few hundred dollars](https://the-decoder.com/google-deepminds-alphaproof-nexus-solves-decades-old-math-problems-for-a-few-hundred-dollars/)：AlphaProof Nexus 自动解决 9 个 Erdős 开放问题，其中 2 个困扰数学家 56 年；系统使用 Lean 编译器验证证明步骤，单题推理成本为数百美元。背景：数学证明是检验模型“可验证推理”的硬场景，比普通问答更难靠语言流畅度蒙混过关。

🔴 **[Model Context Protocol]** [The 2026-07-28 MCP Specification Release Candidate](https://blog.modelcontextprotocol.io/posts/2026-07-28-release-candidate/)：MCP 下一版候选规范发布，核心变化是协议层无状态化，移除 `initialize` 握手和 `Mcp-Session-Id`，并引入扩展、OAuth/OIDC 更一致的授权和正式弃用策略。背景：MCP 已从工具接入协议变成 Agent 基础设施的一部分，无状态化直接影响生产部署、负载均衡和服务端扩展。

🔴 **[36氪 / arXiv]** [SaaS-Bench: Can Computer-Use Agents Leverage Real-World SaaS to Solve Professional Workflows?](https://arxiv.org/html/2605.15777v1)：UniPat AI 的 SaaS-Bench 用 23 个真实开源 SaaS、106 个跨应用任务测试 Computer-Use Agent，最强模型端到端完全通过率也低于 4%。背景：真实办公任务不是单页网页操作，而是长链路、跨系统、带业务状态的执行问题。

## 💡 今日洞察

今天的关键词不是单点模型能力，而是系统可信度。AlphaProof Nexus 和 SaaS-Bench 指向同一个问题：Agent 要进入严肃场景，必须让推理、引用、工具调用和最终结果都可验证。MCP 无状态化、Bumblebee 这类安全扫描、macro-eval 工作流，则是在补 Agent 生产化的基础设施缺口。
